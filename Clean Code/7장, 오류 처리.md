# 오류 처리
- 오류는 언제든지 발생할 수 있다.
- 오류 처리를 하되, 여기저기 흩어진 오류 처리 코드 때문에 원래 로직을 이해하기 어려운 상황은 피해야 한다.


## 오류 코드보다 예외를 사용하라
- 다중 if 문으로 이상 상태를 처리하게 하지 말라.
- 오류가 발생하면 예외를 던지게 하라.


## try-catch-finally 문부터 작성하라
- 예외가 발생해도 호출자가 기대하는 상태를 정의하기 쉬워진다.
- 팁
    1. 강제로 예외를 발생시키는 테스트 케이스를 작성
    2. 테스트를 통과하도록 코드를 작성
